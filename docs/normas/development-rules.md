# Development Rules

Reglas obligatorias del proyecto.

1. Antes de modificar código revisar el archivo actual.
2. Nunca modificar más de lo necesario.
3. Entregar archivo completo si hay cambios.
4. Justificar cada modificación.
5. Mantener coherencia arquitectónica.
6. Evitar soluciones mágicas.    

## Mobile-first

BCC adopta una norma mobile-first explícita.

El usuario principal del panel será mayoritariamente el dueño del negocio, que accederá muchas veces desde móvil.

Toda nueva interfaz debe resolverse primero en móvil y después escalar a desktop.  
Se deben evitar tablas rígidas y priorizar cards, listas, acciones visibles y filtros compactos.
